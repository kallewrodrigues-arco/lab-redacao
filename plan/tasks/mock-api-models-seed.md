# Task: mock-api-models-seed

## Escopo
- **Tipo**: Back-end (BE)

## Objetivo
Definir modelos de dados e seeds cobrindo estados do fluxo (agendada, atrasada, pendente, corrigida, rejeitada; IA pura vs híbrido; rascunho IA vs validada) para suportar o protótipo e o Wizard-of-Oz.

## Entregáveis
- Modelos (mesmo que simples) para: propostas, turmas, alunos, redações, correções IA (rascunho), validações, devolutivas, evolução por competência.
- Seed inicial com dados suficientes para demonstrar todas as telas e estados.

## Critérios de pronto
- É possível demonstrar todos os fluxos sem criar dados manualmente “no banco” a cada demo.
- Existem exemplos para IA pura e híbrido, incluindo transições de status.
